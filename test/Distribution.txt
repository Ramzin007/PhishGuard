Implement content script

Runs on every website.

Collect data:

URL
page title
visible page text
login form detection

Example logic:

const pageData = {
 url: window.location.href,
 title: document.title,
 text: document.body.innerText,
 hasLoginForm: !!document.querySelector("input[type='password']")
}

chrome.runtime.sendMessage({
 type: "SCAN_PAGE",
 data: pageData
})
Implement message receiver

Background script receives scan request.

contentScript → background.js

Example:

chrome.runtime.onMessage.addListener((msg, sender, sendResponse) => {

 if (msg.type === "SCAN_PAGE") {
   const result = analyzeRisk(msg.data)
   sendResponse(result)
 }

})

Member 1 defines data contract used by the entire team.

 text,
 hasLoginForm
}

OUTPUT

{
 riskScore,
 reasons
}

If this contract is not fixed early, integration will collapse.

Member 2 — Domain Intelligence / URL Analyzer

Focus only on URL-based phishing detection.

Files owned
src/background/domainAnalyzer.js
src/utils/levenshtein.js
Feature 1 — Typosquatting detection

Example attacks:

amaz0n.com
paypaI.com
g00gle-security.net

Compare domain with popular domains.

Example list:

amazon.com
paypal.com
google.com
facebook.com
microsoft.com

Use Levenshtein distance.

Example rule:

distance <= 2 → suspicious
Feature 2 — Suspicious keywords in URL

Detect patterns like:

verify-account
secure-login
update-info
account-security

Example:

paypal-security-login.xyz

Add risk score.

Feature 3 — Suspicious TLD

Certain TLDs are heavily abused:

.xyz
.top
.click
.site

Example scoring logic:

typosquatting = +50
suspicious TLD = +20
phishing keywords = +20

Return:

domainRisk = 0–100
Member 3 — Scam Content Detection (NLP)

This member analyzes page content, not URL.

Files owned
src/background/nlpAnalyzer.js
src/content/domExtractor.js
Step 1 — Extract text

Use text already collected by content script.

Step 2 — Detect scam language

Common phishing phrases:

verify your account
urgent action required
security alert
account suspended
limited time
confirm your identity

Simple detection approach:

keyword matching

Example:

if phrase appears → increase risk
Step 3 — Detect login forms

Phishing sites often request credentials.

Check for:

input[type="password"]

Add risk score.

Example scoring:

password form present = +30
scam language detected = +40

Return:

languageRisk

Risk Engine (Integration)

Member 1 integrates all signals.

Create:

riskEngine.js

Example formula:

riskScore =
0.4 * domainRisk +
0.4 * languageRisk +
0.2 * formRisk

Classification:

0–30 safe
30–60 suspicious
60–100 phishing

Return:

{
 riskScore: 72,
 reasons: [
   "Domain similar to paypal.com",
   "Urgent language detected",
   "Login form present"
 ]
}
